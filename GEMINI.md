# GEMINI.md

## Project Overview

This is a Flutter-based mobile application called "FII-Smart" designed for students at the FII faculty of UAIC University in Iasi. The app's primary goal is to help students learn and study course material by providing AI-generated quizzes.

### Key Technologies

*   **Frontend:** Flutter
*   **Backend:** Firebase (Authentication, Firestore, Storage)
*   **AI:** Google Generative AI (Gemini) for quiz generation and content enhancement.

### Architecture

The application follows a service-based architecture, with a clear separation of concerns between the UI (screens), services (business logic), and models (data structures).

*   **`lib/screens`**: Contains the UI for different parts of the application (login, main screen, quiz screen, etc.).
*   **`lib/services`**: Contains the business logic for interacting with backend services like Firebase and Gemini.
    *   `auth_service.dart`: Handles user authentication.
    *   `content_service.dart`: Manages the logic for handling predefined and user-generated content.
    *   `data_service.dart`: Manages data interaction with Firestore.
    *   `gemini_service.dart`: Interacts with the Google Generative AI API to generate quizzes and enhance content.
    *   `storage_service.dart`: Handles file uploads to Firebase Storage.
*   **`lib/models`**: Defines the data structures used in the application (e.g., `UserModel`, `QuizModel`, `UserNote`).

## Building and Running

### Prerequisites

*   Flutter SDK
*   A configured Firebase project with the `google-services.json` (for Android) and `FirebaseApp-Info.plist` (for iOS) files.
*   A `lib/utils/constants.dart` file containing the `geminiApiKey`.

### Running the App

1.  **Get dependencies:**
    ```bash
    flutter pub get
    ```
2.  **Run the app:**
    ```bash
    flutter run
    ```

### Building the App

The project includes a `build_app.sh` script for building the application.

**Usage:**

```bash
./build_app.sh [--debug|--profile|--release] [--apk|--appbundle|--ios|--web]
```

**Examples:**

*   Build a release APK:
    ```bash
    ./build_app.sh --release --apk
    ```
*   Build a debug app bundle:
    ```bash
    ./build_app.sh --debug --appbundle
    ```

## Development Conventions

*   **Linting:** The project uses the `flutter_lints` package to enforce coding standards. The linting rules are defined in `analysis_options.yaml`.
*   **State Management:** The project appears to use `StatefulWidget` and `setState` for state management.
*   **Services:** Business logic is encapsulated in service classes, which are instantiated and used directly in the UI layer.
*   **Data Models:** Data is represented using model classes with `fromJson` and `toJson` methods for serialization.
*   **Quiz Generation:** Quizzes are generated by the `GeminiService`. The service takes context from pre-defined content and user-uploaded notes, combines them, and then generates a quiz.
